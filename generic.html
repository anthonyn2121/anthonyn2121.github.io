CoDrone

As a part of my Introduction to Robotics class I took at Penn State, we were given a CoDrone and were tasked to program the quadrotor to fly a prespecified route. This particular quadrotor is compatible with the Arduino IDE and I took advantage of this to obtain odometry data from the flying tests. This project was not too difficult, as I had previous programming experience with C++ in the Arduino IDE. My drone completed the route with no major difficulties.

This project further gain experience in developing robots and this is my first experience in motion planning.

RRT

This is my first project in graduate school. We previously studied probabilistic roadmap planners that create a path from starting configuration to end configuration by randomly selecting configurations until the path is complete. However, for this lab obstacles were introduced to our robot and we had to come up with a solution to work around the obstacles. The goal of this assignment was to write our own code for the RRT motion planner and then test it on the Lynx robot, a 6DOF robot arm. For future assignments for this course, we will be using the same Lynx robot. To overcome the aforementioned obstacles in our free space, we assume known obstacle positions and create a buffer zone around the obstacles's edges in free space. If our planner randomly sampled a configuration within the buffer zone, our controller would 'discard' the configuration and try again.

The robot was first simulated on MATLAB to ensure that the lynx robot followed a path we expected given the randomly sampled configuration and as well as guaranteed safety to other students and the robot itself. We then uploaded our code to the actual robot to compare the similarities in performance between the actual robot and the simulation.

Waldo Robot

This assignment was for my mechatronics course and the goal of this project was very open ended, I was tasked to create a master-slave system. I was personally inspired by the ISS Remote Manipulator system that consists of a robotic arm to perform station maintenance and perform "costmic catches", as stated by NASA. To complete this project I created two 3DOF manipulators. One would be created with potentiometers as the joints, where I would be able to manipulate my arm myself. This robot would act as the 'master' of the system. The other manipulator would be created with servos, acting as the 'slave'. The brains of this project is the Teensy 2.0.

To interface the Teensy with the potentiometers, the code I wrote reads the potentiometer values sequentially directly from the ADC ports of the microcontroller. This allowed me to get consistent readings from all of the potentiometers, with very little deviations in readings. With these ADC readings, I send a PWM signal to the servos in the slave system that is proportional to the position value of the potentiometers
